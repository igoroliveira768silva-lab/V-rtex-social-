# Vortex-social-

Tecnologias Utilizadas

Backend: PHP (Rodando via servidor embutido no Termux)
Frontend: HTML5, CSS3 (Design responsivo e Dark Mode estilo terminal antigo, tipo hacker)
Interatividade: JavaScript (Manipulação dinâmica de DOM)
Banco de Dados: JSON (Persistência de dados local para alta velocidade de leitura/escrita)
Ambiente de Dev: Termux & Acode

Funcionalidades Principais

Feed em Tempo Real: Sistema de postagens dinâmicas que lê e escreve diretamente em estruturas JSON.
Gestão de Dados Local: Implementação de um CRUD (Create, Read, Update, Delete) sem a necessidade de bancos de dados pesados como MySQL que ainda pretendo aprender, otimizando o uso de hardware mobile.
Interface Otimizada: UI focada na experiência do usuário, com tempos de carregamento reduzidos.

Desafio

Este projeto foi um estudo de caso sobre Arquitetura de Software em Ambientes Limitados. O maior desafio foi integrar o interpretador PHP com a lógica de arquivos JSON para simular um comportamento de banco de dados relacional, garantindo que os dados dos usuários fossem preservados entre as sessões, além de que o ambiente do Android acabou limitando muito o uso de certas funcionalidades.
